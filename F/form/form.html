Тег form

form - набор элементов взаимодействия с клиентом. Как правило используются для отправки считываемой информации на сервер. Например это может быть форма онлайн заказа.

<form action="">
<input type="text" name="username" placeholder="Логин">
<input type="password" name="password" placeholder="Пароль">
<button type="submit"> Отправить </button>
</form>

атрибуты

action=" "

Ссылка на код, обрабатывающий полученые данные. Если адрес не указан страница просто будет обновляться каждый раз при отправке.

method=" "

Метод отправки данных. Существует 2 метода:
get - данные записываются в адресную строку в формате "имя=значение", например email=example@sobaka.rom. Пары "имя=значение" разделяются символом &.
post - посылает на сервер данные в запросе браузера. Это позволяет отправлять большее количество данных, чем доступно методу get.

name=" "

Уникальное имя формы. Позволяет искать форму в скриптах.

autocomplete=" "

Возможность автозаполнения формы ранее сохраненными пользователем данными. Если в браузере отключена возможность автозаполнения атрибут ни на что не повлияет.
on - автозаполнение включено (по умолчанию).
off - отключает автозаполнение.

enctype=" "

Определяет тип кодирования полученных данных.
application/x-www-form-urlencoded - пробелы меняются на + а символы кириллицы будут кодироваться шестнадцатиричным кодом (по умолчанию).
multipart/form-data - для отправки форм с загружаемыми файлами. Данные не кодируются.
text/plain - отправка данных в виде обычного текста, пробелы меняются на + остальное без изменений.

novalidate

Отключение проверки на правильность полученных данных. Например на наличие знака @ при вводе почты.
